# F-1.1. Заголовок Мира (Шаг 1 инициализации [Мира]).

Первый Артефакт инициализации [Мира] (см. [F-1-start.md](F-1-start.md)) — короткое название и одно литературное предложение-аннотация. Ничего, кроме заголовка, на этом шаге не придумывается.

Параллельно этой фичей закладывается общая форма хранения [Артефактов] — все последующие Артефакты будут жить в той же таблице.

## UX-сценарий

1. На странице старта новой [Истории] игрок видит:
   - текстовое поле «Пожелания к миру» (необязательное),
   - набор кнопок-пресетов (фэнтези, мрачное фэнтези, sci-fi и т.д.) — клик подставляет в поле короткую затравку,
   - кнопку «Создать варианты».
2. После нажатия — генерируются ровно **3 варианта** «обложки»: `name` (1–4 слова) + `tagline` (одно предложение).
3. Игрок:
   - выбирает один из трёх → фиксируется `Approved`-артефакт, идём к Шагу 2,
   - либо жмёт «Перегенерировать» (по желанию правит пожелания/пресет).

## Пресеты

| Ключ | Подставляется в поле «Пожелания» |
|---|---|
| `fantasy_high` | Высокое фэнтези: магия, расы, эпический масштаб |
| `fantasy_dark` | Мрачное фэнтези: тяжёлый мир, неоднозначная мораль |
| `scifi_far` | Далёкое будущее: космос, межзвёздные путешествия |
| `cyberpunk` | Киберпанк: корпорации, импланты, плотная урбанистика |
| `post_apoc` | Постапокалипсис: руины цивилизации, дефицит |
| `horror` | Хоррор: давление, страх, неизвестное |
| `detective` | Детектив: расследование, тайна, реалистичный мир |
| `mythic` | Мифы и легенды: древние силы, культурные архетипы |
| `fairytale` | Сказка: волшебный, светлый, для всех возрастов |
| `mundane` | Бытовое, реалистичное, без магии и фантастики |

Пустое поле без пресета = «сюрприз, сам выбери».

## Промпт ИИ

System prompt (фиксирован в коде):

```
Ты — соавтор интерактивной книги-квеста. Сейчас идёт первый шаг
инициализации нового [Мира] для игрока.

ЗАДАЧА: придумать ровно 3 варианта «обложки» мира — короткое
название и одно литературное предложение-аннотацию. Это всего лишь
заголовок, без раскрытия деталей. Локации, население, сюжет — НЕ
придумывай.

ТРЕБОВАНИЯ:
- Названия: 1–4 слова, русский язык, без англицизмов и штампов
  («Хроники X», «Сага о Y» и т.п. — избегай).
- Аннотации: ровно одно предложение, литературное, ёмкое, не более
  240 символов.
- Три варианта должны заметно отличаться по жанру или настроению.
- Учти пожелания игрока, если они даны.

ОТВЕТ: только валидный JSON по схеме. Никаких пояснений и markdown.
```

User-сообщение (шаблон):

```
Пожелания игрока: «{userHint || 'нет, сюрприз'}»
Пресет: {presetKey || 'не выбран'}
```

JSON-схему передаём в `format` параметра Ollama API:

```json
{
  "type": "object",
  "properties": {
    "options": {
      "type": "array", "minItems": 3, "maxItems": 3,
      "items": {
        "type": "object",
        "properties": {
          "name":    { "type": "string", "minLength": 1, "maxLength": 60 },
          "tagline": { "type": "string", "minLength": 1, "maxLength": 240 }
        },
        "required": ["name", "tagline"],
        "additionalProperties": false
      }
    }
  },
  "required": ["options"],
  "additionalProperties": false
}
```

## Доменные модели (quest.db)

```csharp
public enum WorldStatus    { Initializing = 1, InProgress = 2, Finished = 3 }
public enum ArtifactKind   { WorldHeader = 1, SettingLevels = 2, Population = 3,
                             Backstory = 4, StoriesAndTasks = 5 }
public enum ArtifactStage  { Initialization = 1, Enrichment = 2, Update = 3 }
public enum ArtifactStatus { Draft = 1, Approved = 2, Rejected = 3, Superseded = 4 }

public sealed class World
{
    public Guid Id { get; set; }
    public string? Title { get; set; }          // null до Approved-WorldHeader
    public WorldStatus Status { get; set; }
    public DateTimeOffset CreatedAt { get; set; }
    public ICollection<Artifact> Artifacts { get; set; } = new List<Artifact>();
}

public sealed class Artifact
{
    public Guid Id { get; set; }
    public Guid WorldId { get; set; }
    public ArtifactKind Kind { get; set; }
    public ArtifactStage Stage { get; set; }
    public int Version { get; set; }            // 1+ для Approved, 0 для Draft
    public ArtifactStatus Status { get; set; }
    public string PayloadJson { get; set; } = ""; // jsonb в PG, типизуется на C#
    // трассировка генерации:
    public string? Model { get; set; }
    public string? Prompt { get; set; }         // итоговый промпт (system+user в одной строке)
    public string? RawResponse { get; set; }
    public int? PromptTokens { get; set; }
    public int? CompletionTokens { get; set; }
    public int? DurationMs { get; set; }
    public DateTimeOffset CreatedAt { get; set; }
}
```

Индексы:
- `Artifact (WorldId, Kind, Status)` — основной запрос «последний Approved такого-то Kind».
- `Artifact (WorldId, CreatedAt DESC)` — лента генераций для отладки.

## Payload в Artifact.PayloadJson

Структура зависит от `Kind`. Для `WorldHeader`:

**Draft** (после генерации, до выбора):
```json
{
  "userHint": "...",
  "preset":   "fantasy_dark" | null,
  "options":  [{"name":"…","tagline":"…"}, …]
}
```

**Approved** (после выбора игрока):
```json
{
  "name":            "…",
  "tagline":         "…",
  "userHint":        "…",
  "preset":          "fantasy_dark" | null,
  "rejectedOptions": [{"name":"…","tagline":"…"}, …]
}
```

## Жизненный цикл артефакта

```
[ Draft v=0 ]  ← один на каждый клик «Сгенерировать»
     │
     │  игрок выбрал один из 3 → создаётся Approved
     ▼
[ Approved v=1 ]
     │
     │  игрок передумал, перегенерировал и выбрал другой
     ▼
[ Approved v=2 ]   prev Approved → Superseded
```

Все Draft того же `Kind` при создании Approved помечаются `Superseded`. Все предыдущие `Approved` того же `Kind` тоже становятся `Superseded` — каноничной считается версия с максимальным `Version` и `Status=Approved`.

## API (REST, JSON)

| Метод | Маршрут | Тело | Ответ |
|---|---|---|---|
| `POST` | `/api/worlds` | — | `{ id, status, createdAt }` |
| `POST` | `/api/worlds/{worldId}/header/generate` | `{ userHint, preset, model? }` | `{ draftId, options:[{name,tagline}×3], generationMs }` |
| `POST` | `/api/worlds/{worldId}/header/{draftId}/approve` | `{ chosenIndex }` | `{ approvedId, version, name, tagline }` |
| `GET`  | `/api/worlds/{worldId}` | — | `{ id, title, status, header? }` (header = последний Approved) |

`model` опционален — если не передан, берётся `Ollama:DefaultModel`. Имя модели должно быть в `Ollama:Models` (валидируется), иначе 400.

Ошибки модели (некорректный JSON, schema violation после ретрая) → 502 с диагностикой.

## UI (минимум)

`Views/Worlds/Start.cshtml`:
- форма с textarea «Пожелания» + пресет-чипы + select модели (из `Ollama:Models.*`) + кнопка «Создать варианты»;
- блок «Варианты» (появляется после ответа): три карточки `name + tagline + кнопка "Выбрать"`;
- кнопка «Перегенерировать»;
- после `approve` — редирект на `/Worlds/{id}` (заглушка-страница со списком артефактов, расширим в F-1.2).

Скрипт — пока vanilla JS (`fetch`), потому что одна страница. TypeScript-сборка отложена до момента, когда фронта станет ощутимо больше.

## Что НЕ в этой фиче

- Шаги 2+ ([Уровни Сеттинга], [Население], …) — отдельными спеками.
- Оценка артефактов моделью-критиком — отдельная фича.
- Стриминг ответа Ollama — пока не нужен, генерация заголовка короткая.
- Аутентификация/мульти-игрок — нет.
- Список миров, удаление, переименование — позже.
