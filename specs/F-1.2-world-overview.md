# F-1.2. Общее описание [Мира] (Шаг 1 Сеттинга).

После approve заголовка (F-1.1) ИИ автоматически генерирует **общее описание [Мира]** — фундамент Сеттинга, на который опираются все следующие шаги инициализации.

## Аннотация ≠ Описание Мира

- **Аннотация** (F-1.1, tagline) — «задняя обложка книги», видима игроку, хук без спойлеров.
- **Описание Мира** (F-1.2) — содержательное описание устройства мира для ИИ-ведущего. Игрок его не видит (в проде). Это «мозг» мира.

Пример (для post_apoc, scale=compact, fates=[survival, mystery]):

| | Текст |
|---|---|
| **Аннотация** (F-1.1) | Ты просыпаешься в темноте, где каждый вдох — борьба, а каждый шаг — загадка. |
| **Описание** (F-1.2) | Действие происходит в подземном бункере исследовательского комплекса «Сигма-7», построенного перед ядерной войной. Бункер расположен на глубине 200 метров, рассчитан на 500 человек, но сейчас обитаемая зона сократилась до трети. Четыре сектора — жилой, технический, лабораторный и командный — соединены центральным коридором. Системы жизнеобеспечения работают с перебоями, запасы продовольствия критически низкие. За пределами бункера — радиоактивная пустошь. Связь с другими убежищами отсутствует третий год. |

## Что генерируется

Один артефакт с текстовым описанием всего [Мира]. Объём не фиксирован — ИИ определяет сам в зависимости от Scale: от одного предложения для `room` до нескольких абзацев для `cosmic`. Ограничение только снизу — описание должно быть содержательным enough, чтобы служить контекстом для генерации уровней, населения и предыстории.

## Артефакт

Каждый артефакт имеет:

| Поле | Описание | Пример |
|---|---|---|
| **Kind** | Тип артефакта | `World`, `WorldPartL1`, `WorldPartL2`, … |
| **Name** | Название на естественном языке (русском) | «Корабль Ностромо» |
| **ArtifactId** | Структурированный идентификатор: `Kind[.ParentSlug].ThisSlug` | `World.ShipNostromo` |

**Формирование ArtifactId:**
- ИИ генерирует английский PascalCase-slug для каждого артефакта.
- Корневой артефакт мира: `World.<Slug>` (например `World.ShipNostromo`).
- Дочерние артефакты: `Kind.<ParentSlug>.<ThisSlug>` (например `WorldPartL1.ShipNostromo.CapitanCockpit`).
- По ArtifactId видно: (а) что это за артефакт (Kind), (б) чей он ребёнок, (в) как называется.

Для данного шага:
- Kind = `World`
- Name = название мира на русском (из F-1.1, например «Последний свет»)
- ArtifactId = `World.<Slug>` (slug генерирует ИИ, например `World.LastLight`)

## Когда

Сразу после approve заголовка — автоматически, без участия игрока. Игрок видит прогресс на странице Show.

## Контекст для промпта

ИИ получает:
- название + аннотация (из F-1.1),
- оси: preset, fates, pacing, scale (из World),
- пожелания игрока (userHint из DraftPayload заголовка).

Scale определяет масштаб — ИИ описывает мир соответствующего размера.

## Хранение

Один артефакт: `Kind = World`, `Stage = Initialization`, `Status = Approved` (сразу, без Draft — игрок не выбирает).

Payload:
```json
{
  "description": "Действие происходит в подземном бункере..."
}
```

## Отображение

На странице Show — в разделе «Сеттинг» (для разработки; в проде сеттинг скрыт от игрока).

## Что НЕ в этой фиче

- Уровни локаций 2+ (части мира) — следующий подшаг F-1.3.
- Население, предыстория, сюжет — отдельные фичи.
- Оценка моделью-критиком — в следующих этапах.
