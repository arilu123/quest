## Идея

Книга-квест с ИИ-ведущим. Игрок и ИИ ходят по очереди: ИИ описывает сцену, игрок отвечает в свободной форме. Перед самим квестом ИИ создаёт [Мир] — но важно разделять два понятия:

- **Аннотация** — «задняя обложка книги», видимый игроку текст (название + хук). Это лицо мира.
- **Сеттинг** — внутреннее описание мира, которое ИИ создаёт **для себя** (локации, население, предыстория, сюжетные направления). Игрок его не видит. Это мозг мира.

Инициализация мира — это последовательная генерация Сеттинга (заголовок → локации → население → предыстория → сюжетные направления).

## Что уже работает

- **Шаг 1: Заголовок [Мира].** На странице старта игрок задаёт пожелания и делает четыре выбора: **стиль**, **тон судьбы** (0–2), **темп завязки** и **масштаб** (комнатный/малый/средний/крупный/эпический — влияет на глубину и длительность). ИИ предлагает 3 варианта «обложки». Игрок выбирает один.
- **Шаг 2 (автоматический): общее описание [Мира].** После выбора заголовка ИИ в фоне пишет «мозг мира» — что это за место, как устроено. Игрок этот текст не видит.

## Главные решения по концепту

1. **[Мир] создаётся многоэтапно.** Не одной большой генерацией, а блоками: заголовок → локации → население → предыстория → сюжетные направления. Так у ИИ меньше галлюцинаций и меньше шансов «забыть» что-то важное.
2. **Каждый шаг = выбор из нескольких вариантов.** Игрок не принимает первое попавшееся, а выбирает из трёх (или может перегенерировать). Это превращает создание мира в часть игрового опыта.
3. **Четыре оси выбора на старте.** Игрок выбирает: (а) **стиль** — жанровый пресет, (б) **тон судьбы** — 0–2 типа переживания (два смешиваются), (в) **темп завязки** — точка входа в историю, (г) **масштаб** — размер мира (комнатный → эпический), который определяет число уровней локаций и длительность квеста. Все выборы подмешиваются во все промпты.
4. **Разные шаги — разные модели.** Для лёгких текстовых шагов (типа заголовка) — быстрая модель (10 с на 3 варианта). Для сложных структурных шагов (дерево локаций, население) подключим модель помассивнее. Это компромисс между скоростью итераций и качеством.
5. **Принцип «один запрос — один артефакт».** ИИ не строит за раз ни дерево локаций, ни иерархию персонажей. Приложение знает, что уже создано, передаёт это в контексте и просит ровно один следующий артефакт. Так все локальные модели справляются с задачей одинаково надёжно — а у нас всегда есть полный контроль над структурой мира.
6. **История версий сохраняется.** Все предложенные ИИ варианты (даже отвергнутые) остаются в базе — это позволит позже сравнивать модели, отлаживать промпты и при желании привлекать вторую модель-критика для оценки качества.
7. **MVP — русский язык, один игрок, локально.** Без мультиязычности и пользователей — это потом.

## Что дальше

- 🟡 **Уровни локаций (F-1.3).** Следующий шаг. ИИ автономно генерирует локации каждого уровня — по одной за вызов. Количество уровней = Scale (1–5). Открытый вопрос — сколько локаций на каждом уровне и кто определяет «достаточно».
- ⏳ Дальше по инициализации — всё автономно по той же стратегии (один артефакт за вызов): население → предыстория → план «главы».
- ⏳ После инициализации — **игровой цикл**: ход ИИ ↔ ход игрока. ИИ должен контролировать правдоподобность ответов игрока и дополнять [Мир] по ходу истории.
- ⏳ Поздние идеи: список созданных миров, продолжить/удалить, оценка артефактов второй моделью-критиком.
