# Status

Краткая концептуальная сводка. Если вернёшься к проекту через месяц — чтобы быстро вспомнить, что задумали и куда идём.

## Идея

Книга-квест с ИИ-ведущим. Игрок и ИИ ходят по очереди: ИИ описывает сцену, игрок отвечает в свободной форме. Перед самим квестом ИИ создаёт [Мир] (заголовок, локации, население, предысторию, сюжетные направления) — это и есть **текущая фаза разработки**.

## Что уже работает

- **Шаг 1: Заголовок [Мира].** На странице старта игрок задаёт пожелания и делает три выбора: **стиль** (жанровый пресет), **тон судьбы** (0–2 типа переживания — например «загадки + выживание», они смешиваются) и **темп завязки** (точка входа в историю). ИИ предлагает 3 варианта «обложки» — название и аннотацию в стиле задней обложки книги. Игрок выбирает один.

## Главные решения по концепту

1. **[Мир] создаётся многоэтапно.** Не одной большой генерацией, а блоками: заголовок → локации → население → предыстория → сюжетные направления. Так у ИИ меньше галлюцинаций и меньше шансов «забыть» что-то важное.
2. **Каждый шаг = выбор из нескольких вариантов.** Игрок не принимает первое попавшееся, а выбирает из трёх (или может перегенерировать). Это превращает создание мира в часть игрового опыта.
3. **Три оси выбора на старте.** Игрок выбирает не только антураж, но и (а) тип переживания — **тон судьбы** (можно выбрать 0–2 из: приключения, моральный выбор, тайны, драма, выживание, камерная история; два тона смешиваются) и (б) точку входа — «темп завязки» (сразу в экшн / прямо в момент / незадолго до событий / плавное развитие / издалека). Все выборы подмешиваются во все промпты и задают настроение всему квесту. Сама аннотация написана в стиле задней обложки книги — хук без спойлеров.
4. **Разные шаги — разные модели.** Для лёгких текстовых шагов (типа заголовка) — быстрая модель (10 с на 3 варианта). Для сложных структурных шагов (дерево локаций, население) подключим модель помассивнее. Это компромисс между скоростью итераций и качеством.
4. **История версий сохраняется.** Все предложенные ИИ варианты (даже отвергнутые) остаются в базе — это позволит позже сравнивать модели, отлаживать промпты и при желании привлекать вторую модель-критика для оценки качества.
5. **MVP — русский язык, один игрок, локально.** Без мультиязычности и пользователей — это потом.

## Что дальше

- 🟡 **Уровни локаций.** Следующий шаг инициализации [Мира]: ИИ придумывает структуру локаций — от макроуровня (вселенная / планета / город / квартира) до стартовой точки. На обсудить: генерируем всё дерево разом или каскадом по уровням.
- ⏳ Дальше по инициализации: население (народы и персонажи, включая скрытых) → предыстория → план «главы».
- ⏳ После инициализации — **игровой цикл**: ход ИИ ↔ ход игрока. ИИ должен контролировать правдоподобность ответов игрока и дополнять [Мир] по ходу истории.
- ⏳ Поздние идеи: список созданных миров, продолжить/удалить, оценка артефактов второй моделью-критиком.
